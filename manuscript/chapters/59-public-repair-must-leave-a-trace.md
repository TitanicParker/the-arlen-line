## Formal Note L
### Public Repair Must Leave a Trace

When the instrument enters public use, correction must not be hidden.

A private notebook may silently replace one answer with another. A public instrument cannot. If a classification changes, the change must leave a trace. Otherwise repair becomes indistinguishable from erasure. The reader sees only the new answer and cannot learn what failed, what survived, or why the route changed.

Public repair must leave a trace.

This trace need not be theatrical. It should not become confession, drama, or self-defense. It should be simple:

The prior classification treated the visible word as dominant.

The revised classification restores sentence-level landing.

Closure type remains Does.

Orientation changes from Toward to Across.

Operator changes from Relation to Correction.

The invariant moves from one cell to another.

The reason is this eliminated candidate was reinstated.

This is how correction becomes transmissible.

A repaired answer that leaves no trace teaches obedience to the latest output. A repaired answer that names the route teaches method. It lets the reader see that error is not outside the instrument. Error is one of the ways the instrument learns where its own procedure must be more exact.

This matters for human trust.

People who have been corrected without trace often experience the correction as disappearance. The earlier offering is gone. The earlier reasoning is treated as if it never existed. The authority simply now says otherwise. Institutions do this constantly. They revise, reframe, supersede, close, or uphold without preserving the route by which the change occurred.

The archive must not imitate that.

If it changes a classification, it should say what changed.

If it changes a section, it should preserve the backup.

If it changes a source interpretation, it should name the source pressure that required the change.

If it changes a workflow, it should explain what failure the new rule prevents.

This is not clutter. It is lawful memory.

Of course, not every minor edit belongs in the body of the book. The manuscript need not record every typo, backup, or drafting pass. But the principle must govern the intellectual work: where a structural claim changes, the repair should be legible.

This is especially important in teaching. A learner should see examples of wrong classification repaired. Not only perfect receipts. Perfect receipts can make the system seem magical and the learner’s uncertainty shameful. Repaired receipts show that the method is walkable. They show how to return, how to reinstate a candidate, how to retest an axis, how to compute again.

The public corpus should therefore include some failures.

Not failures as spectacle.

Failures as route.

A sentence first misread by local feature. A sentence first mistaken as Relation when Correction dominates. A sentence first treated as Closure when Integration is enough. A sentence whose orientation was misread because the content was plural but the landing converged. These cases teach more than polished correctness alone.

The model also requires traced repair. When the model revises an answer, it should not say merely, “You are right.” It should identify the failed gate. It should say whether the failure was sentence replacement, unlicensed elimination, axis mismatch, operator dominance, registry mismatch, or unstable reread. This gives the user a way to trust correction without trusting apology.

Trace is the difference between apology and repair.

Apology may soothe.

Repair restores route.

In the manuscript workflow, the backup system is the material form of this law. Each append pass preserves a prior state. The working document moves forward, but the previous body is not destroyed. The flat copy is refreshed, but the backup keeps return available. This is not merely technical prudence. It is the archive obeying its own theory.

Public repair also prevents doctrine. A doctrine hides its mistakes or reinterprets them as misunderstandings. An instrument records where it failed and how it was corrected. It does not need to appear immaculate. It needs to remain answerable.

This may make the public form less elegant.

That is acceptable.

A repaired path is more trustworthy than a flawless facade.

The reader does not need perfection. The reader needs a way back.
